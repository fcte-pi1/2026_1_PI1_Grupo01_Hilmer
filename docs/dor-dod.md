# Definição

A Definição de Preparado, ou Definition of Ready (DoR), estabelece critérios que indicam se um requisito está preparado para ser adicionado ao backlog da Sprint. Enquanto que a Definição de Pronto, ou Definition of Done (DoD), define os parâmetros para que aquela atividade da Sprint esteja pronta para ser apresentada na Sprint Review. 

## DoR
1. O requisito possui informação necessária para ser trabalhado?
 
2. O requisito cabe em uma sprint?
 
3. O requisito, caso correspondente ao software, está representado por uma história de usuário?

4. O requisito possui critérios de aceitação suficientes, referentes aos pacotes de trabalho necessários para cada área envolvida?

5. Se houver, as dependências do requisito estão identificadas?
    
## DoD
1. Entrega um incremento do produto?

2. Os critérios de aceitação estão contemplados?

3. Os testes de integração estão documentados?

4. Os arquivos adicionados ou alterados não desrepeitam a organização de nenhuma dessas pastas: docs, hw, mec, src(backend, firmware, frontend)?

5. Existe algum arquivo maior do que 5MB que está sendo enviado ao repositório?

### Considerações ao DoD

- Para o critério 4, verifique as instruções presentes no README.md da pasta em que está sendo trabalhada.
  
- Se o critério 5 for verdadeiro, armazene o arquivo em outra plataforma e coloque aqui apenas o link de direcionamento externo.
