# Definição
    A Definição de Preparado, ou Definition of Ready (DoR), estabelece critérios que indicam se um requisito está preparado para ser adicionado ao backlog da Sprint. Enquanto que a Definição de Pronto, ou Definition of Done (DoD), define os parâmetros para que aquela atividade da Sprint esteja pronta para ser apresentada na Sprint Review. 

## DoR
    - O requisito possui informação necessária para ser trabalhado?
    - O requisito cabe em uma sprint?
    - O requisito, caso correspondente ao software, está representado por uma história de usuário?
    - O requisito possui critérios de aceitação suficientes, referentes aos pacotes de trabalho necessários para cada área envolvida?
    - Se houver, as dependências do requisito estão identificadas?
    
## DoD
    - Entrega um incremento do produto?
    - Os critérios de aceitação estão contemplados?
    - Os testes de integração estão documentados?
